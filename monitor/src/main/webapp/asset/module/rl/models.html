Reward  r(s, a)
r = −( λ · E + (1−λ) · T )
State: Observable cluster metrics (power, utilization, params)
Action: 9-dim parameter vector Δθ (freq, concurrency, heap…)
Environment: Hadoop cluster returns st+1 after executing the config
Reward: Weighted negative of energy & runtime; λ controls the trade-off
Reward Function
r(s, a)  =  − (  λ · E(s,a)  +  (1−λ) · T(s,a) )
E: Normalised energy
= (E − Emin) / (Emax − Emin)
T: Normalised runtime
= (T − Tmin) / (Tmax − Tmin)
λ = 0.10: Energy priority
90% weight on energy saving
λ = 0.30: Performance priority
70% weight on runtime speedup
